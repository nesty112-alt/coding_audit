# coding_audit
[Python/Tkinter] SQLite와 Pandas로 만드는 GUI 데이터 유효성 검증 프로그램 (업무 자동화)


[사용 설명서] 데이터 검증 프로그램 (Data Validation App)

본 프로그램은 원본 데이터(CSV)를 불러와 사용자가 정의한 검증 규칙(Excel)의 
SQL 쿼리를 기반으로 데이터의 오류를 자동 검출하고 리포트를 생성하는 프로그램입니다.

----------------------------------------------------------------------
1. 프로그램 실행 전 사전 준비 사항
----------------------------------------------------------------------
프로그램을 정상적으로 사용하기 위해서는 아래의 파일들이 미리 준비되어야 합니다.

■ 데이터 파일 (.csv)
  - 검증 대상이 되는 원본 데이터입니다.
  - Main 데이터와 Sub 데이터로 나누어 각각 또는 동시에 검증할 수 있습니다.

■ 검증 규칙(룰) 파일 (.xlsx)
  - 데이터의 어떤 부분을 검증할지 정의한 엑셀 파일입니다.
  - 엑셀 파일의 첫 번째 행(헤더)에는 반드시 아래의 컬럼이 포함되어야 합니다.
    * Rule_Name : 검증 규칙의 이름 (예: 고객번호 누락 검증)
    * SQL_Query : 오류 데이터를 추출할 SQL 쿼리문
    * number    : (선택사항) 검증 규칙의 번호
  - [중요] SQL 쿼리 작성 시 주의사항:
    * Main 데이터를 검증할 때 FROM 절의 테이블명은 반드시 'main'으로 작성해야 합니다.
      (예: SELECT * FROM main WHERE customer_id IS NULL)
    * Sub 데이터를 검증할 때 FROM 절의 테이블명은 반드시 'sub'으로 작성해야 합니다.
      (예: SELECT * FROM sub WHERE amount < 0)

■ 결과 저장 폴더
  - 검증 완료 후 오류 내역 리포트(엑셀 파일)가 저장될 컴퓨터 내의 폴더입니다.


----------------------------------------------------------------------
2. 화면 구성 및 핵심 기능
----------------------------------------------------------------------
프로그램 창은 크게 3개의 영역으로 나누어져 있습니다.

① 파일 및 경로 설정 영역 (상단)
  - Main/Sub 데이터 파일, 검증 규칙 파일, 결과 저장 폴더의 경로를 지정합니다.
  - 우측의 [찾아보기] 버튼을 눌러 편리하게 파일을 선택할 수 있습니다.
  - 한 번 입력한 경로들은 프로그램 종료 후에도 자동으로 기억되어, 다음 실행 시 
    다시 입력할 필요가 없습니다. (config.json 파일에 저장됨)

② 실행 옵션 및 버튼 영역 (중간)
  - 어떤 데이터를 검증할지 선택합니다. (Main만 실행 / Sub만 실행 / 모두 실행)
  - 경로 설정 영역에 파일 경로를 입력하면, 프로그램이 입력 상태를 감지하여 
    올바른 실행 옵션을 자동으로 선택해 줍니다.
  - 우측의 [▶ 검증 실행] 버튼을 누르면 작업이 시작됩니다.

③ 실행 결과 대시보드 영역 (하단)
  - 진행 상황과 검증 결과를 실시간으로 보여주는 표(테이블)입니다.
  - 상태값 표시 안내:
    * ✅ 정상     : SQL 쿼리 실행 결과, 조건에 맞는 오류 데이터가 0건인 경우
    * 🚨 오류     : 조건에 일치하는 오류 데이터가 1건 이상 발견된 경우
    * ⚠️ SQL실패  : SQL 문법 오류 등으로 인해 검증 쿼리가 실행되지 못한 경우


----------------------------------------------------------------------
3. 단계별 상세 사용 방법
----------------------------------------------------------------------
[1단계] 파일 및 폴더 지정하기
  - 프로그램 상단의 [찾아보기] 버튼을 클릭하여 준비한 Main/Sub 데이터와 
    검증 룰 파일을 각각 선택합니다.
  - 맨 아래 [결과 저장 폴더]의 [찾아보기]를 눌러 결과물이 저장될 폴더를 선택합니다.

[2단계] 검증 실행하기
  - 실행 옵션(라디오 버튼)이 내가 원하는 대로 설정되었는지 확인합니다.
  - [▶ 검증 실행] 버튼을 클릭합니다.
  - 실행 중에는 버튼이 [⏳ 실행 중...]으로 변경되며, 프로그램이 멈춘 것처럼 
    보여도 백그라운드에서 정상 작동 중이므로 잠시 기다려 주세요.

[3단계] 결과 확인 및 유용한 기능 활용하기
  - 버튼이 다시 [▶ 검증 실행]으로 돌아오면 검증이 완료된 것입니다.
  - 대시보드 표에서 오류 건수를 확인합니다.
  - [마우스 더블클릭] : 표의 항목을 더블클릭하면 상세 메시지 창이 열립니다. 
    특히 '⚠️ SQL실패'인 경우 어떤 부분에서 에러가 났는지 상세히 확인할 수 
    있으며, 하단의 [메시지 복사하기] 버튼으로 에러 내용을 복사할 수 있습니다.
  - [마우스 우클릭] : '비고(오류 메시지)' 칸을 마우스 우클릭하면 팝업창 없이 
    즉시 에러 메시지를 클립보드에 복사할 수 있는 메뉴가 나타납니다.

[4단계] 결과 엑셀 리포트 확인하기
  - 지정한 결과 저장 폴더로 이동하면 아래와 같은 파일이 생성되어 있습니다.
    * Main 실행 시: main_audit_result.xlsx
    * Sub 실행 시 : sub_audit_result.xlsx
  - 오류가 발견된 경우: '오류내역조회' 시트에 오류 데이터가 모두 모여 있으며, 
    맨 첫 번째 열에 [오류유형]이 추가되어 어떤 룰에서 발생한 오류인지 쉽게 
    필터링하여 볼 수 있습니다.
  - 오류가 없는 경우: '오류내역_없음' 시트에 정상 메시지가 기록됩니다.


----------------------------------------------------------------------
4. 자주 묻는 질문 (FAQ) 및 문제 해결
----------------------------------------------------------------------
Q. [▶ 검증 실행] 버튼을 눌렀는데 "결과를 저장할 폴더를 지정해주세요"라고 뜹니다.
A. 맨 아래에 있는 '결과 저장 폴더'가 비어있거나 올바르지 않은 경로일 때 발생하는 
   메시지입니다. [찾아보기] 버튼을 눌러 결과 엑셀 파일이 저장될 폴더를 꼭 선택해 주세요.

Q. 결과 대시보드에 '⚠️ SQL실패'가 많이 뜹니다. 원인이 무엇인가요?
A. 대부분 엑셀 룰 파일에 작성된 SQL 쿼리문과 원본 CSV 데이터의 컬럼명이 일치하지 
   않거나, 테이블명을 잘못 지정했을 때 발생합니다. 해당 항목을 더블클릭하여 에러 
   메시지를 확인한 후, SQL 쿼리문이나 테이블명(main 또는 sub)을 수정해 주세요.

Q. 검증 룰 엑셀 파일의 헤더 이름을 규칙(Rule_Name) 대신 '룰이름' 등으로 바꿔도 되나요?
A. 안 됩니다. 프로그램 내부적으로 'Rule_Name'과 'SQL_Query'라는 영문 이름으로 
   컬럼을 찾도록 고정되어 있으므로, 반드시 해당 명칭을 유지해야 합니다.
